[ 입출력 ] : 입력(컴퓨터 안으로) 과 출력(컴푸터 밖으로)
[ 파일 ] : 컴퓨터에 쓰이는 데이터들을 모아놓은 것
[ 스트림 ] : 컴퓨터내 데이터/자료 가 다니는 통로
    - 데이터가 다니는 통로인 스트림은 바이트 단위로 이동한다.
    - 자바(JVM : 자바 가상메모리) <-- 스트림(데이터/바이트) --> 윈도우(C:)
        //JVM(휘발성/프로그램 종료시 초기화)
        //파일처리(비휘발성 / 영구저장)
    - 영문 / 특수문자 1글자당 : 1바이트, 한글 1글자당 : 2바이트

[ 바이트 ] : 컴퓨터가 처리 하는 용량 단위
    1. 비트 : 0 또는 1
    2. 바이트 : (8개 bit -> 1 byte)
    3. 1024byte -> 1KB -> MB -> KB
    주로 영문(아스키코드 기반) 은 1바이트, 한글(유니코드-JAVA)은 2바이트

[ 파일 입출력 ] : 파일에 데이터를 입력 하고 출력하는 작업
    - 파일 입출력 과 관련된 클래스 / 인터페이스
    - 사용이유 : 자바에서 다루는 데이터를 영구 저장하기 위해서 자바 외부에 저장하는 기술.
                (로그처리)

    1.(C)FileOutputStream 객채변수명 = new FileOutputStream("파일경로") -->일반 예외 ( 예외 필수 )
        1.write(출력할바이트배열) : 현재 연동된 파일에 쓰기(출력) 하는 함수

    2.(C)FileInputStream 객채 변수명 = new FileInputStream("파일경로") --> 일반 예외( 예외 필수) IO
        1. .read(바이트배열) : 파일의 내용을 바이트 배열로 읽어오기
        2. new String(바이트 배열) : 바이트 배열을 문자열로 반환 해주는 객체 생성자

[ 파일 관련 클래스]
    -File 클래스
        1. .isFile() : 지정한 경로와 파일이 존재하는지 여부 반환 함수.
        2. .get() : 지정한 파일의 이름 반환 함수
        3. .length() : 지정한 파일의 용량(바이트) 반환 함수
        4. .exists() :  지정한 경로의 파일이 존재하는 여부 t/f 반환 함수
        5. .createNewFile() : 지정한 경로의 파일 생성 하는 함수, 성공시 T, 실패시 F

[ 경로 ]
    1. 절대 경로 : 지정한 파일의 위치한 경로를 모두 작성한 경우
    D:/user/kor2024_2/src/day25/test.txt
    2. 상대 경로 : 지정한 파일의 위치한 경로를 *현재 기준*으로 작성한 경우
    ./ : 현재 위치 뜻한다
    ../ : 상위 위치 뜻한다.
    (D:/user/kor2024_2)생략 =  ./src/day25/test1.txt


[ 기타 참고 함수]
    "문자열". getbytes(); : 문자열을 바이트 배열로 반환 해주는 함수

-객체를 사용하는 이유: 메소드(멤버함수) 사용하기 위해서
-객체없이 메소드를 사용 할수 있나 static 이면 가능하다.
-메소드는 왜 사용하는가. 자바회사에서 미리 제공한 코드들의 집합(라이브러리)
 중 일종인 메소드를 사용하면 빠른 개발이 가능하니까.

    - 종류 : 1.일반예외 2.실행예외
        1. 일반예외 ( 컴파일 / 코드번역 과정에서 코드를 검사하는 예외)

        2. 실행예외 ( 컴파일 / 코드번역 과정에서 코드를 검사하지 않는 예외)
            -대부분/주로 : NullPointerExcetion 등등
